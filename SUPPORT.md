# Support

Use GitHub issues for reproducible defects and scoped feature requests. Before
opening an issue, search existing issues and run the relevant checks from
[CONTRIBUTING.md](CONTRIBUTING.md).

Include the VoxLattice version or commit, Python version, operating system,
Docker and NVIDIA runtime versions, GPU name, sanitized configuration, expected
behavior, actual behavior, and the smallest reproduction possible. Never post
tokens, certificates, private keys, raw audio, or personal data.

Community support is best effort and has no guaranteed response time. For
suspected vulnerabilities, do not use public support channels; follow
[SECURITY.md](SECURITY.md).
